# YouTube 영상 요약(학습용) 가이드 문서 (Markdown 템플릿)

> 이 문서는 **영상 + 자막(Transcript)** 을 기반으로 “학습 가능한 형태”로 구조화된 요약을 만들기 위한 표준 가이드입니다.  
> 목표: **핵심 주제/개념을 논리적으로 재구성**하고, **숫자 기반 인사이트**와 **탐구형 질문**까지 포함해 “복습·전이(transfer) 가능한 노트”를 만든다.

---

## 최상단 요약(작성자가 먼저 채우는 5줄)

- **한 문장 결론:** (이 영상이 궁극적으로 주장하는 바)
- **핵심주제 3개:** (1) … (2) … (3) …
- **가장 중요한 숫자/근거 3개:** (예: %, $, 시간, 샘플 수, 전후 비교)
- **적용 시나리오 2개:** (현업/학습에 어디에 쓰나)
- **탐구형 질문 3개:** (반박/확장/검증 질문)

---

## Step-by-step 작성 프로세스 (Tree of Thought: 3명의 전문가 협업)

#### Step 1) 입력 확보 & 전처리 (Expert A: 리서처)
- **필수 입력**
	- 영상 링크(URL)
	- 전체 재생시간(YouTube 표시 시간)
	- 자막(자동/수동) 또는 스크립트 전문
- **전처리 체크리스트**
	- 자막의 오타/구어체 정리(의미 훼손 없이 최소 수정)
	- 용어 통일(동일 개념의 다른 표현을 하나로)
	- “예시/농담/반복”을 표시해 두고, 요약에서는 중요도에 따라 축약  
- 참고(Transcript/Transcript analysis 개념): citeturn0search0turn0search1

#### Step 2) 구조 설계(학습용) (Expert B: Instructional Designer)
- **노트 구조는 ‘재학습/자기점검’이 가능한 형태**가 유리
	- Cornell 노트처럼 “핵심 질문/단서 + 본문 + 요약” 구조를 차용하면 복습 효율이 높음 citeturn0search8turn0search16turn0search12
- **요약은 주제 중심으로 묶어 일관성을 만든다(테마/전환점/반복 신호 활용)** citeturn0search3

#### Step 3) 논리적 요약 + 검증 질문 생성 (Expert C: 분석가)
- **학습 효과를 높이려면**
	- “설명해보며 빈칸 찾기(가르치듯 설명)” 방식으로 개념 이해도를 점검(Feynman) citeturn0search5turn0search9
	- 질문은 Bloom의 인지 수준(이해→적용→분석→평가→창조)로 점진적으로 난이도를 올려 설계 citeturn0search6turn0search18

> 협업 규칙(간단):  
> - A는 “사실/근거”를, B는 “학습 구조”를, C는 “논리·검증”을 책임진다.  
> - 서로의 결과를 1회씩 교차 리뷰하고, **근거 없는 일반화/숫자 없는 주장**은 C가 제거한다.

## 최종 출력 포맷(필수 섹션)

## 1) 제목
- **[영상 제목]**  
- (선택) 채널명 / 업로드 날짜 / 링크

## 2) 전체 재생시간
- **총 재생시간:** `HH:MM:SS`
- (선택) 분석에 사용한 자막 언어/버전

## 3) 전체 요약 (학습용 “구조화 요약”)
#### 3.1 핵심주제 (Key Themes)
- 주제 1: (문장형)
	- 왜 중요한가(문제/맥락)
	- 영상이 제시한 해법/주장
	- 근거(사례/데이터/비유)  
- 주제 2: …
- 주제 3: …

#### 3.2 핵심개념 (Key Concepts)
- 개념 A: 정의(한 줄) → 작동원리(2~3줄) → 오해하기 쉬운 포인트(1줄)
- 개념 B: …
- (권장) **개념 간 관계도**
	- A → B로 이어지는 이유(인과/조건/상호작용)
	- 반례/예외조건

#### 3.3 숫자 기반 인사이트 (Quant & Evidence)
- 숫자/지표 1: `값` / 출처(영상 내 발화 시점 또는 자료명)
	- 의미: 무엇을 보여주나?
	- 비교: 전후/대안/베이스라인(가능하면)
	- 한계: 샘플, 가정, 일반화 범위  
- 숫자/지표 2: …
- 숫자/지표 3: …

#### 3.4 탐구형 질문 (Inquiry Questions)
- **이해/확인(Understand)**
	- “왜 ~라고 했지?”, “조건이 바뀌면 어떻게 되지?”
- **적용/전이(Apply)**
	- “내 업무/학습 상황에 적용하면 어떤 입력값이 필요하지?”
- **분석/비판(Analyze/Evaluate)**
	- “이 주장에 필요한 전제는?”, “대안 가설은?”
- **창조(Create)**
	- “이 내용을 기반으로 실험/프로젝트를 설계한다면?”  
- 질문 설계 프레임(참고: Bloom’s): citeturn0search6turn0search18

## 4) 구간별 요약 (타임라인 기반)

> 원칙: **구간은 3~7개**가 보통 가독성이 좋다(영상 길이에 따라 조정).  
> 각 구간은 “무슨 얘기 → 핵심근거(숫자/사례) → takeaway(한 줄)”로 끝낸다.

#### 구간 템플릿
- `00:00–05:12` **[구간 제목]**
	- 핵심주제: …
	- 핵심개념: …
	- 숫자/근거: …
	- takeaway(한 줄): …
	- 탐구형 질문(1~2개): …

- `05:12–12:40` **[구간 제목]**
	- …

(구간 반복)

## 품질 체크리스트(완성 직전 3분 점검)

- **논리성**
	- 주장 → 근거 → 결론이 연결되는가?
	- 예시(사례)가 결론을 “증명”하는지, 단지 “설명”하는지 구분했는가?
- **근거성**
	- “숫자/조건/범위”가 없는 문장이 과도하게 많은가? (있다면 보완)
- **학습성**
	- 핵심개념이 “내 말로 설명 가능”한 수준인가? (Feynman 점검) citeturn0search5
	- 질문이 단순 암기형에 치우치지 않았는가? (Bloom 레벨 상향) citeturn0search6turn0search18
- **가독성**
	- 1문단 1아이디어, 불필요한 수식어 제거
	- 구간별 요약이 타임라인을 따라가며 겹치지 않는가?
